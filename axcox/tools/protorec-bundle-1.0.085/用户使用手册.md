# ProtoRec 用户使用手册

## 欢迎使用

这份手册是给日常使用 ProtoRec 的同学准备的。

你只需要完成两件事：

- 安装浏览器插件
- 在项目里启动本地服务

完成后，就可以通过浏览器插件抓取当前页面，并把结果保存到项目里。

## 你会看到哪些文件

解压压缩包后，目录大致如下：

```text
protorec-bundle-1.0.085/
  browser-extension/
    PM-yuanxing-picker/
  project-local-service/
    protorec-local-service/
  README.md
  用户使用手册.md
```

其中：

- `browser-extension/PM-yuanxing-picker`：浏览器插件
- `project-local-service/protorec-local-service`：本地服务

## 使用前准备

开始前请先确认：

- 电脑已安装 Chrome 浏览器
- 电脑已安装 Node.js 18 或更高版本
- 电脑可以正常使用 `npm`

如果不确定 Node.js 是否已安装，可在终端执行：

```bash
node -v
npm -v
```

如果能看到版本号，就表示环境正常。

## 第一步：安装浏览器插件

请按以下步骤操作：

1. 打开 Chrome 浏览器
2. 进入扩展程序页面：`chrome://extensions/`
3. 打开右上角“开发者模式”
4. 点击“加载已解压的扩展程序”
5. 选择 `browser-extension/PM-yuanxing-picker`

安装完成后，你会在浏览器右上角看到 ProtoRec 插件图标。

## 第二步：把本地服务放到项目里

建议每个项目放一份独立的本地服务，这样不同项目的抓取结果不会混在一起。

请把下面这个目录整体复制到你的项目根目录：

```text
project-local-service/protorec-local-service
```

复制完成后，项目结构示例：

```text
你的项目/
  protorec-local-service/
  其他项目文件...
```

## 第三步：启动本地服务

### macOS

双击：

```text
protorec-local-service/start.command
```

### Windows

双击：

```text
protorec-local-service/start.bat
```

首次启动时，程序会自动安装依赖，可能需要等待一会儿，这是正常现象。

## 第四步：确认服务已启动

本地服务启动后，请在浏览器打开以下任意地址：

- `http://localhost:3001/`
- `http://localhost:3001/health`

如果页面返回一段 JSON 数据，表示服务已经启动成功。

## 第五步：开始抓取页面

服务启动成功后，按下面步骤使用：

1. 打开你要抓取的目标页面
2. 点击浏览器右上角的 ProtoRec 插件
3. 在插件里点击抓取或同步按钮
4. 等待抓取完成

## 抓取结果保存在哪里

抓取和还原结果会保存在项目里的以下目录：

```text
protorec-local-service/workspace/pages
protorec-local-service/workspace/temp_proto
protorec-local-service/workspace/quality
```

其中：

- `workspace/pages`：还原后的页面代码
- `workspace/temp_proto`：抓取过程中的中间文件
- `workspace/quality`：质量检查结果

## 常见问题

### 双击启动没反应

请优先检查：

- 是否已安装 Node.js
- 是否允许系统执行脚本
- 是否被安全软件拦截

### 提示端口被占用

这表示本机的 `3001` 端口已经被其他程序占用。

遇到这种情况，请联系分发方协助处理。

## 推荐使用方式

为了使用更稳定，建议统一按以下方式操作：

1. 每个项目复制一份 `protorec-local-service`
2. 每次抓取前先启动本地服务
3. 抓取完成后到 `workspace/pages` 查看结果

## 你只需要记住

如果你是普通使用者，只需要关注这两部分：

- 浏览器插件
- 本地服务

除了这两部分外，不需要额外安装其他组件。
