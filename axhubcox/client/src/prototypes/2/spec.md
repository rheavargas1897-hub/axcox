# 车载设备组合售卖管理原型规格

## 页面定位

该原型由 Google AI Studio 项目转换而来，目标是在 Axhub Make 中运行一套车载设备组合售卖管理后台，覆盖套餐、质保、提成、安装部位和字典配置等核心运营配置流程。

## 页面结构

- 顶部导航：营运通品牌、业务中心导航、搜索框、通知/设置入口、账号资料。
- 左侧导航：销售中心、商品中心、报表管理、系统设置四组菜单。
- 主内容区：根据左侧菜单切换不同业务模块。
- 浮层/详情区：用于套餐、质保、提成等对象的新建、编辑、查看与复制。

## 功能模块

- 套餐管理：列表查询、新建套餐、查看详情、编辑入口，展示套餐状态、版本、安装部位数、生效时间、业务类型、终端类型、销售/质保/提成方案等字段。
- 质保管理：列表、新建、查看、编辑、复制，支持展示绑定适用信息。
- 质保方案配置模块一：首次购买设置；从系统车型列表中逐行选择车型，每个车型独立配置是否赠送质保。若不赠送，可继续配置是否支持加购，以及支持加购时的加购费用。
- 质保方案配置模块二：监控服务配置，仅套餐质保展示；从系统车型列表中逐行选择车型，每个车型独立配置监控服务类型、监控费用、续费年费价格、是否与套餐服务年限挂钩。
- 质保方案配置模块三：延保配置；从系统车型列表中逐行选择车型，每个车型独立配置是否支持延保、固定延保费用；仅套餐质保展示是否与套餐服务年限挂钩。
- 提成管理：列表、新建、查看详情、编辑、复制，支持安装提成与销售提成方案配置。
- 安装部位管理：展示安装部位配置列表。
- 信息配置表：维护业务字典类配置。

## 交互说明

- 点击左侧「套餐管理 / 质保管理 / 提成管理 / 信息配置表」切换主内容页面。
- 点击「新建套餐」打开套餐创建流程。
- 点击列表操作中的「编辑 / 查看详情」进入对应浮层或详情态。
- 质保和提成模块支持复制已有方案，并以副本形式进入编辑态。
- 在质保方案中，模块一按车型逐行维护首次购买设置：选择“是否赠送质保”为“否”后展示“是否支持加购”；选择支持加购后填写该车型的加购费用，单位为元/年。
- 在套餐质保中，模块二按车型逐行维护监控服务：选择“收费”后填写监控费用；填写续费年费价格；选择“与套餐年限挂钩”为“是”后表达套餐服务年限到期后不可继续续费监控服务的业务约束。
- 在延保配置中，选择“支持延保”为“是”后填写固定延保费用；仅当质保类型为套餐质保时，展示并配置是否与套餐服务年限挂钩。
- 顶部搜索、筛选、分页按钮作为原型交互控件展示。

## 样式规范

- 技术栈：React + TypeScript + Tailwind CSS V4。
- 主色：蓝色/靛蓝色，用于当前导航、主按钮和关键操作。
- 背景：白色管理后台工作台，边框使用浅灰分隔。
- 信息密度：偏高密度 B 端表格布局，优先保证字段可扫描性。
- 图标：使用 `lucide-react`。
