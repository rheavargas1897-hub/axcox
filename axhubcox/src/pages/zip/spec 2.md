# 设备库存统计仪表盘

## 📋 业务与功能

### 1.1 核心目标
> 提供企业设备库存的实时统计与查询视图，通过“仓库视图 / 产品视图 / 设备明细”三种视角帮助用户快速定位库存状态、设备流转和维保信息。

该页面适合作为库存管理系统的数据看板，支持按仓库、按产品以及设备层级展开分析，帮助运营人员快速判断库存健康度与设备流向。

### 1.2 功能清单
> 列出所有功能模块及其优先级

- **顶部导航与导出**：展示页面标题、时间范围、导出按钮
- **视图切换**：仓库视图 / 产品视图 / 设备明细三种视图切换
- **筛选条件**：支持按仓库筛选、关键字搜索（编号/名称/车牌号）
- **仓库视图**：按仓库汇总各类库存状态，点击展开显示该仓库的产品明细
- **产品视图**：按产品汇总库存状态，点击展开显示该产品相关设备明细
- **设备明细视图**：展示设备列表，包含状态、来源、成本、所属等信息

### 1.3 交互要点
> 关键的交互触发点、反馈机制、状态变化

- 点击“按仓库统计 / 按产品统计 / 设备明细”按钮：切换视图
- 选择仓库下拉框：仅显示所选仓库的数据（其余视图也会受影响）
- 输入搜索关键词：实时过滤当前视图的数据列表
- 点击表格行（仓库/产品）：展开/折叠对应的“嵌套明细”区域
- 点击“导出数据”按钮：触发数据导出（当前为 UI 按钮，可扩展行为）

## 📊 内容规划

### 2.1 信息架构
> 模块划分、信息层级、内容组织方式

```
设备库存统计仪表盘
├── 头部（标题 + 时间范围 + 导出按钮）
├── 控制区
│   ├── 视图切换按钮
│   ├── 仓库筛选下拉
│   ├── 搜索输入框
│   └── 过滤/设置按钮
├── 内容区（数据表）
│   ├── 仓库视图（可展开仓库下的产品）
│   ├── 产品视图（可展开产品下的设备）
│   └── 设备明细视图
```

### 2.2 数据来源
> 数据源优先级：用户提供数据 > 项目数据表 > 生成示例数据

- **数据类型**：设备库存记录（Equipment）
- **数据源**：内置示例数据（`src/pages/zip/src/mockData.ts`）
- **关键字段**：
  - `id`: 设备/记录唯一 ID
  - `sn`: 设备序列号
  - `warehouseId`: 所属仓库 ID
  - `warehouseName`: 仓库名称
  - `productId`: 产品 ID
  - `productName`: 产品名称
  - `status`: 设备状态（可用/占用/冻结/在途/已出库/待维修/维修中/待回收/已借出）
  - `inboundTime`: 入库时间（用于筛选当前月份）
  - `outboundTime`: 出库时间（可选）
  - `purchaseCost`: 采购成本

### 2.3 内容示例
> （可选）重要的示例内容、文案语气、术语规范

- **视图切换**：按钮文案保持简洁，例如“按仓库统计”“按产品统计”“设备明细”
- **字段展示**：库存量、可用/占用/冻结/在途等字段采用不同颜色区分状态，提高可读性

## 🎨 布局与结构

### 3.1 整体布局
> 布局模式（单栏/双栏/网格/自由）、模块尺寸、比例约束

- **布局模式**：单栏 + 卡片式数据表（整体最大宽度 7xl，容器居中）
- **容器宽度**：流式宽度（最大 `max-w-7xl`），内边距 `px-4 sm:px-6 lg:px-8`
- **关键尺寸**：
  - 头部高度：固定 64px（`h-16`）
  - 表格行高度：约 44px（`py-3`）

### 3.2 响应式适配
> （如适用）断点、适配策略

- **桌面端（≥1200px）**：表格全宽展示，导航与筛选并排
- **平板端（768-1199px）**：维持表格可滚动，控制区垂直排列
- **移动端（<768px）**：当前页面主为桌面表格，大屏展示优先；若需要可进一步优化为卡片列表（暂未实现）
