# 车载设备组合售卖场景页面结构与后台原型清单

## 1. 文档目标

本文档基于《车载设备组合售卖场景套餐管理功能 PRD》进一步拆解页面结构与后台原型范围，用于指导产品原型设计、前后端任务拆分与评审。

## 2. 信息架构总览

### 2.1 管理端一级导航

- 套餐中心
- 销售开单
- 订单中心

## 3. 管理端页面结构

### 3.1 套餐中心

#### 3.1.1 套餐列表页

页面目标：查看所有套餐主体，并进入版本管理与配置。

核心内容：

- 套餐名称
- 套餐 ID
- 适用客户类型
- 可见渠道
- 当前上架状态
- 当前生效版本
- 最近更新时间

核心操作：

- 新建套餐
- 搜索/筛选
- 上下架
- 查看详情
- 进入版本列表

原型优先级：P0

#### 3.1.2 新建/编辑套餐页

页面目标：维护套餐主体基础信息，不处理具体版本内容。

核心内容：

- 套餐名称
- 套餐说明
- 适用客户类型
- 渠道可见性
- 套餐主体状态

核心操作：

- 保存草稿
- 提交保存
- 返回列表

原型优先级：P0

#### 3.1.3 套餐详情页

页面目标：展示套餐主体信息、历史版本、销售与履约概况。

核心内容：

- 基础信息卡片
- 当前生效版本摘要
- 历史版本时间轴/列表
- 关联订单数
- 续费中的服务数

核心操作：

- 创建新版本
- 查看版本详情
- 停用套餐主体

原型优先级：P1

#### 3.1.4 版本列表页

页面目标：按套餐查看所有版本及其状态。

核心内容：

- 版本号
- 状态
- 生效时间
- 失效时间
- 变更说明
- 创建人/发布时间

核心操作：

- 从当前版本复制
- 查看版本详情
- 发布版本
- 停用版本
- 归档版本

原型优先级：P0

#### 3.1.5 版本编辑页

页面目标：完成套餐版本配置。

建议分区：

- 基础信息区
- 套餐项配置区
- 规则配置区
- 价格配置区
- 服务项配置区
- 生效设置区
- 校验结果区

核心内容：

- 版本号与来源版本
- 变更说明
- 必选设备项配置
- 选配服务项配置
- 加购配件配置
- 数量上下限
- 依赖/互斥/必选规则
- 硬件价、年费价
- 生效时间设置

核心操作：

- 保存草稿
- 校验配置
- 发布
- 定时生效
- 放弃编辑

原型优先级：P0

#### 3.1.6 版本详情页

页面目标：以只读方式查看某个版本快照。

核心内容：

- 版本基础信息
- 套餐项快照
- 价格快照
- 规则快照
- 变更记录
- 关联订单列表入口

核心操作：

- 复制为新版本
- 导出版本快照

原型优先级：P0

### 3.2 销售开单

## 6. 原型绘制建议

### 6.1 优先绘制顺序

1. 套餐列表页
2. 版本列表页
3. 版本编辑页
4. 线下开单页
5. 订单详情页
6. 出库/发货处理页
7. 车辆绑定处理页
8. 售后详情处理页
9. 续费处理页
10. 用户端套餐详情页与下单确认页

### 6.2 原型表现建议

- 版本编辑页建议采用分区卡片或分步表单，避免单页过长
- 订单详情页建议采用页签结构，承载费用、履约、服务、售后、日志
- 售后详情处理页建议突出审批流与协商结果留痕
- 线下开单页建议在“指定版本”处增加权限提示和风险提示
- 用户端套餐详情页需突出“硬件费 + 年费 + 可选服务”的组合展示

## 7. 页面与功能映射关系

| 功能模块 | 管理端页面 | 用户端页面 |
|---|---|---|
| 套餐管理 | 套餐列表、套餐详情、新建/编辑套餐 | 套餐列表、套餐详情 |
| 版本管理 | 版本列表、版本编辑、版本详情 | 无直出版本页，以下单命中逻辑隐式使用 |
| 线下销售 | 线下开单列表、线下开单页 | 无 |
| 订单管理 | 订单列表、订单详情 | 我的订单列表、订单详情 |
| 履约交付 | 履约任务、出库发货、车辆绑定、服务开通/安装工单 | 订单详情、车辆设备列表、我的服务 |
| 售后管理 | 售后单列表、售后申请创建、售后详情处理 | 售后申请、售后记录 |
| 续费管理 | 续费列表、续费处理页 | 续费列表、续费支付页 |
| 基础配置 | SKU 管理、服务项管理、权限配置、操作日志 | 无 |
